# 异常处理

## 一、异常的概念

在Python中，异常是指在程序执行期间出现的错误或异常情况。如果异常未被捕获和处理，程序将终止执行并将错误信息显示给用户。因此，了解和处理异常是编写可靠且健壮的Python代码的重要部分。

程序停止执行并且提示错误信息这个动作，我们通常称之为：抛出(raise)异常

## 二、常见异常类型

1. SyntaxError：语法错误。例如拼写错误、缺少括号等。
2. TypeError：类型错误。例如将不同类型的变量进行操作等。
3. ValueError：数值错误。例如将字符串转换为数值类型时出现错误等。
4. NameError：变量未定义。例如访问未定义的变量等。
5. IndexError：索引越界。例如访问数组越界等。
6. KeyError：键不存在。例如访问字典中不存在的键等。
7. ZeroDivisionError：除零错误。例如将一个数除以零等。

## 三、捕获异常

在Python中，可以使用try...except...finally语句来处理异常。try语句块用来包含可能会抛出异常的代码，except语句块用来处理异常，finally语句块则用来执行清理工作，无论是否出现异常都会执行。具体语法如下：

```python
try:
    # 可能会抛出异常的代码
except ExceptionType1:
    # 处理异常类型1的代码
except ExceptionType2:
    # 处理异常类型2的代码
...
else:
    # 如果没有出现异常，则执行这里的代码
finally:
    # 无论是否出现异常都会执行这里的代码
```

在使用try语句块捕获异常时，可以使用except语句块指定要捕获的异常类型。如果出现指定的异常类型，则执行对应的except语句块。如果没有指定异常类型，则捕获所有异常类型。

## 四、抛出 raise 异常

在Python中，`raise`关键字用于手动抛出异常。当代码遇到一个问题时，Python会自动抛出异常，但有时如果我们想在代码的某个地方手动引发异常，就可以使用`raise`关键字。

以下是一个简单的示例，演示如何使用`raise`引发一个异常：

```python
def divide_by_zero():
    try:
        x = 5 / 0
    except ZeroDivisionError:
        raise Exception("Cannot divide by zero.")

divide_by_zero()
```

在这个示例中，`divide_by_zero`函数尝试将`5`除以`0`，这会导致Python引发`ZeroDivisionError`异常。在`except`块中，我们使用`raise`引发通用异常，并将消息设置为“Cannot divide by zero.”。最终，该函数抛出了一个异常，这将停止函数执行并将控制权交给调用方。

## 五、assert 断言

在Python中，`assert`语句用于断言某个条件是否为真。如果断言的条件为 `False`，则会引发异常，否则程序将正常执行。

使用`assert`语句可以帮助我们快速地发现代码中的错误和缺陷，从而提高代码的可靠性和健壮性。

assert语句的语法格式如下：

```python
assert condition, error_message
```

其中，condition是要检查的条件，如果为False，则抛出异常；error_message是可选的错误信息，可以在抛出异常时同时输出错误信息。

下面是一个简单的示例，演示了如何使用assert语句：

```python
def divide(x, y):
    assert y != 0, "division by zero"
    return x / y

print(divide(10, 2))    # 输出5.0
print(divide(10, 0))    # 抛出AssertionError异常，并输出错误信息"division by zero"
```

在上面的代码中，我们定义了一个函数divide，用于计算x除以y的结果。在函数中，我们使用assert语句检查了y是否为0，如果为0，则抛出AssertionError异常，并输出错误信息"division by zero"。这样，我们就可以在程序的某个位置，明确地检查某个条件是否为真，从而避免潜在的错误情况。

注意，`assert`语句仅在检查条件为 `False`时才会引发异常，并且可以使用可选的错误消息参数来提供更具体的错误信息。此外，在开发和调试代码时，您可以使用Python的`-O`命令行选项来禁用`assert`语句，从而忽略所有断言失败。

## 六、自定义异常

在Python中，我们可以通过定义一个新的异常类来自定义异常。自定义异常类必须是已有异常类Exception的子类。

以下是一个简单的示例，演示如何创建和使用自定义异常：

```python
class MyException(Exception):
    def __init__(self, message):
        super().__init__(message)

def divide(x, y):
    if y == 0:
        raise MyException("Cannot divide by zero.")
    return x / y

a = 10
b = 0

try:
    result = divide(a, b)
except MyException as e:
    print("Caught my custom exception:", e)
else:
    print(result)
```

在这个示例中，我们创建了一个名为`MyException`的自定义异常类，该类继承自内置的`Exception`类，并实现了一个构造函数来初始化异常消息。

然后我们定义了一个`divide`函数，该函数将`x`除以`y`，但在`y`为0的情况下会引发我们的自定义异常`MyException`。

最后，我们进行了一次除零运算，并使用`try...except`块和我们自定义异常来捕获和处理异常。如果我们捕获了自定义异常，则输出包含异常消息的消息“Caught my custom exception: Cannot divide by zero.”。否则，如果 `try` 语句块执行成功，则输出计算结果。

通过创建自定义异常类，我们可以更好地组织和管理我们的代码，并使代码更易于阅读和理解。
