## 1、概述

### （1）多任务

什么叫“多任务”呢？简单地说，就是操作系统可以同时运行多个任务。打个比方，你一边在用浏览器上网，一边在听MP3，一边在用Word赶作业，这就是多任务，至少同时有3个任务正在运行。还有很多任务悄悄地在后台同时运行着，只是桌面上没有显示而已。

现在，多核CPU已经非常普及了，但是，即使过去的单核CPU，也可以执行多任务。由于CPU执行代码都是顺序执行的，那么，单核CPU是怎么执行多任务的呢？

答案就是操作系统轮流让各个任务交替执行（`时间片轮转`），任务1执行0.01秒，切换到任务2，任务2执行0.01秒，再切换到任务3，执行0.01秒……这样反复执行下去。表面上看，每个任务都是交替执行的，但是，由于CPU的执行速度实在是太快了，我们感觉就像所有任务都在同时执行一样。

真正的并行执行多任务只能在多核CPU上实现，但是，由于任务数量远远多于CPU的核心数量，所以，操作系统也会自动把很多任务轮流调度到每个核心上执行。

### （2）并发和并行

并发（Concurrency）和并行（Parallelism）是计算机科学中两个重要的概念，它们描述了任务执行的方式，但是有着明显的区别。

**并发**：并发是指多个任务在**逻辑**上同时发生。这不一定意味着它们在物理上同时发生，即并发任务可能会交替执行。在单核 CPU 上，多个任务会通过任务切换来实现并发执行，即每个任务执行一段时间后切换到另一个任务。这种方式在逻辑上看起来像是同时执行的，因为切换速度非常快。

例如，在 Python 中，可以使用线程（Threading）或协程（Coroutine）来实现并发。并发是适用于 I/O 密集型任务的一种有效方式，因为在等待 I/O 操作（如网络请求，读写文件）完成的时候，CPU 可以切换到另一个任务。

**并行**：并行是指多个任务在**物理**上同时发生。这需要多个处理器（或多核 CPU）来同时执行多个任务。在并行执行中，多个任务可以真正意义上的同时执行，而不需要进行任务切换。

例如，在 Python 中，可以使用进程（Process）来实现并行执行。并行执行是一种有效的方式来加速 CPU 密集型任务，因为可以同时使用多个 CPU 核心。

总结一下，这两个概念的主要区别在于，“并发”描述的是任务的组织方式和调度方式，“并行”描述的是任务的执行方式。并发关注的是程序在宏观上的表现，而并行关注的是程序在微观上的运行状态。

### （3）实现多任务的方式

Python是一种面向对象、解释性、高级程序语言，可进行多种编程方式，例如面向对象、函数式、模块化等，因此具有较高的灵活性。

Python支持多任务实现，它可以通过以下方式实现多任务处理：

1. 多进程（multiprocessing）：它可以让一个程序中的同时执行多个进程，每个进程都运行在其自己的Python解释器进程中，Python提供了multiprocessing模块来实现多进程任务。
2. 多线程（Threading）：Python中提供的threading模块可以实现多线程处理，它可以同时执行多个线程，每个线程共享程序的内存空间，因此线程间可以直接传递数据和共享数据。
3. 协程（Coroutine）：在Python 2.5版本引入了生成器(generator)，可以用作协程的基础，Python 3.5版本引入了asyncio模块，使用async和await关键字实现协程。
4. 异步编程（Asynchronous Programming）：通过异步编程实现非阻塞式I/O操作。

总之，Python提供了多种方式来实现多任务处理，开发人员可以根据需求选择适合的方式来进行多任务处理。

## 2、全局解释器锁

全局解释器锁（GIL）是Python解释器的一个重要组成部分。它的主要目的是保护Python的内存管理系统，防止在多线程环境中产生竞争条件。简单地说，全局解释器锁确保了任何时刻只有一个线程在执行Python字节码。

### （1）为什么存在GIL？

Python语言本身并不是线程安全的，因为它的内存管理器没有为多线程环境设计。为了防止多个线程同时访问同一个Python对象，导致内存不一致，Python解释器引入了全局解释器锁来同步所有的Python字节码执行。

### （2）GIL如何影响多线程编程？

在单核CPU环境中，GIL的影响并不明显，因为即使在没有GIL的情况下，CPU也只能同时运行一个线程。

然而，在多核CPU环境中，GIL就成了一个问题。理论上，多核CPU可以让每个核运行一个线程，从而实现真正的并行。但是由于GIL的存在，任何时刻只有一个线程在执行Python字节码，即使在多核CPU环境中，Python的多线程也只能实现并发，而非并行。

这就意味着，在CPU密集型的任务中，Python的多线程并不能充分利用多核CPU的优势，因此在这种情况下，通常使用多进程或其他并行方案（如Cython、NumPy、multiprocessing模块等）会更有效。

### （3）GIL的影响是全局的吗？

GIL主要影响的是Python的多线程执行。其他的并发或并行方案，如多进程或异步IO，并不受GIL影响。

需要注意的是，GIL只是CPython（Python的官方和最常见的实现）的一个特性。一些其他的Python实现，如Jython或IronPython，就没有GIL。

### （4）Python是否会移除GIL？

GIL的存在确实给Python的并行编程带来了一些限制，但它的移除并不容易。因为GIL的存在使得Python的内存管理变得简单，安全，并且大多数Python代码都依赖于这种行为。

因此，尽管已经有一些提议和尝试去移除GIL，但到目前为止，它仍然是CPython的一个重要部分。

在Python的多线程编程中，理解GIL及其对线程行为的影响是非常重要的。在设计多线程应用时，你需要考虑GIL，并根据你的具体需求选择最合适的并发或并行方案。

## 3、进程、线程、协程对比

Python进程，线程，协程都是Python中用于实现多任务的机制，但它们之间有很多不同点。

- 进程：是系统进行资源分配和调度的一个独立单位。进程是一个正在运行中的程序的实例。它拥有自己的内存空间、系统资源和打开的文件等。在多进程编程中，每个进程都是相对独立的，进程间通信需要使用IPC（进程间通信）机制。进程启动和切换的开销比较大，适合用于CPU密集型任务，如图像处理、计算密集型的任务等。

  比如 在一台电脑上能够同时运行多个QQ

  一个程序至少有一个进程，一个进程至少有一个线程。

- 线程：是进程的一个实体，是CPU调度和分派的基本单位，它是比进程更小的能独立运行的基本单位.线程自己基本上不拥有系统资源，只拥有一点在运行中必不可少的资源(如程序计数器，一组寄存器和栈)，但是它可与同属一个进程的其他的线程共享进程所拥有的全部资源。线程是进程内的一个独立执行流程，一个进程可以有多个线程。同一进程中的所有线程共享相同的内存空间和系统资源。在多线程编程中，线程间通信可以直接访问共享内存，但需要加锁保证线程安全。线程启动和切换的开销比进程小，适合用于IO密集型任务，如网络编程等。

  比如 一个QQ中的多个聊天窗口

- 协程：协程是一种轻量级的线程，协程也是在同一个线程中运行的。协程可以在代码中任意切换执行流程，而不需要操作系统进行切换。协程之间的切换只需要保存当前的上下文，而不需要切换内核栈，因此协程启动和切换的开销比线程小得多。协程适合用于IO密集型任务，如异步编程等。

选择进程、线程、协程需要根据任务的特点和性质进行选择。通常，IO密集型任务适合使用协程，而CPU密集型任务适合使用进程或线程。

**请仔细理解如下的通俗描述**

- 有一个老板想要开个工厂进行生产某件商品（例如剪子）
- 他需要花一些财力物力制作一条生产线，这个生产线上有很多的器件以及材料这些所有的为了能够生产剪子而准备的资源称之为：进程
- 只有生产线是不能够进行生产的，所以老板的找个工人来进行生产，这个工人能够利用这些材料最终一步步的将剪子做出来，这个来做事情的工人称之为：线程
- 这个老板为了提高生产率，想到3种办法：
  1. 在这条生产线上多招些工人，一起来做剪子，这样效率是成倍増长，即**单进程 多线程**方式
  2. 老板发现这条生产线上的工人不是越多越好，因为一条生产线的资源以及材料毕竟有限，所以老板又花了些财力物力购置了另外一条生产线，然后再招些工人这样效率又再一步提高了，即**多进程 多线程**方式
  3. 老板发现，现在已经有了很多条生产线，并且每条生产线上已经有很多工人了（即程序是多进程的，每个进程中又有多个线程），为了再次提高效率，老板想了个损招，规定：如果某个员工在上班时临时没事或者再等待某些条件（比如等待另一个工人生产完谋道工序，之后他才能再次工作） ，那么这个员工就利用这个时间去做其它的事情，那么也就是说：如果一个线程等待某些条件，可以充分利用这个时间去做其它事情，其实这就是：协程方式

**简单总结**

1. 进程是资源分配的单位
2. 线程是操作系统调度的单位
3. 进程切换需要的资源很最大，效率很低
4. 线程切换需要的资源一般，效率一般（当然了在不考虑GIL的情况下）
5. 协程切换任务资源很小，效率高
6. 多进程、多线程根据cpu核数不一样可能是并行的，但是协程是在一个线程中所以是并发
